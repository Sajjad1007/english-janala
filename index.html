Frequently Asked Questions

01. What is the difference between var, let and const in Javascript?

In JavaScript, var, let and const differ in scope, hoisting and mutability. var is function-scoped, can be re-declared, updated and is hoisted with an initial value of undefined, which can cause unexpected behavior. let is block-scoped, cannot be re-declared but can be updated and is hoisted without initialization, causing a ReferenceError if accessed before declaration. const is also block-scoped, cannot be re-declared or updated and must be initialized at declaration, though objects and arrays declared with const can have their properties modified.

02. What is the difference between map(), forEach() and filter() in Javascript?

In JavaScript, map(), forEach() and filter() are array methods with different purposes. map() iterates over an array, transforms each element and returns a new array with modified values, making it useful for data transformation. forEach() also iterates over an array but does not return a new array, making it ideal for performing actions like logging or updating the UI. filter() is used to create a new array containing only elements that satisfy a given condition, making it useful for extracting subsets of data. As like map(), forEach() and filter() do not modify the original array.

03. How do arrow functions differ from regular functions?

Arrow functions in JavaScript (=>) provide a shorter syntax compared to regular functions and do not bind their own this, instead inheriting it from the surrounding scope. This makes them useful for callbacks and simple functions but problematic for object methods and event handlers where this needs to reference the calling object. Unlike regular functions, arrow functions also do not have an arguments object, requiring the use of rest parameters (...args) instead. Regular functions are dynamically bound to the calling object, making them better suited for methods inside objects. While arrow functions are great for concise and functional programming, regular functions should be used when explicit this binding or arguments is needed.

04. How do promises work in JavaScript?

In JavaScript, promises are used to handle asynchronous operations by representing a value that might be available now, later or never. A promise has three states: pending (initial state), fulfilled (operation successful, executing .then()) and rejected (operation failed, executing .catch()). Promises allow chaining with .then() for sequential execution and .catch() for error handling, making code more readable and manageable. For example, fetch() returns a promise, allowing us to process API responses asynchronously. Instead of using callbacks, promises provide a cleaner way to write asynchronous code and they work seamlessly with async/await for even more readability.

05. How do closures work in JavaScript?

In JavaScript, closures occur when a function remembers and can access variables from its outer scope, even after the outer function has executed. This happens because inner functions retain a reference to the variables of their parent function, allowing data to persist. For example, in a function factory or a counter, closures help maintain private state without using global variables. They are widely used for data encapsulation, event handling and maintaining state in asynchronous operations. By leveraging closures, JavaScript enables more modular, memory-efficient and reusable code, making them a powerful feature in functional programming.
